# Visual Audit Checklist

Use this file when a task needs a tighter review pass instead of broad direction.

## Theme

- Is the screen using the dark MecaniaOS shell consistently?
- Are text, border, and focus contrast levels readable?
- Are colors semantic instead of decorative?

## Layout

- Is the primary action obvious?
- Is the page structured into panels with clear hierarchy?
- Is there wasted space that could be converted into useful context?

## Components

- Are cards, inputs, buttons, tabs, and badges visually consistent?
- Are radii, spacing, and panel heights aligned?
- Are hover, active, disabled, and selected states obvious?

## Domain fit

- Does the screen feel like workshop software rather than a generic admin?
- Is vehicle, client, or order context visible when needed?
- Are states like reception, process, approval, and delivery easy to scan?

## Polish

- Are empty states integrated into the same visual language?
- Does mobile preserve hierarchy and action clarity?
- Are there any leftover light-theme elements breaking the screen?
